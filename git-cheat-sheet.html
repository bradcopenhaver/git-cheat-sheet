Git commands used on Day 1.

  • Git Setup and Config

    • git init : initializes working directory as a git repository
    • git pair __ __ : using intials from .pairs file, attaches names and emails to project directory

  • Git Committing

    • git add ____ : adds named file to be committed
    • git-pair-commit -m "___" : commits working directory to git with message describing changes

  • Git Info

    • git log : lists commit history for the current branch
    • git status : Lists the files where changes have been made to be committed.
    • git about : Lists the information about the currently assigned pair's names.

  • Git Remote

    • git remote add [remote name] [remote URL] : Sets up a new remote location with a name and location.
    • git remote -v : Shows the names and URLs for all of the remote repositories that the project's Git repository has stored.
    • git push [remote name] [branch name] : Copies the code to the remote repository from the local Git repository.
    • git clone [remote URL] : Copies the code and commit history from a remote repository to a local repository.
